배열을 이용한 리스트 구현

배열의 프린팅

배열을 실제로 출력하면 각 원소의 메모리 주소를 출력

배열의 원소 프린팅 : 반복문을 이용하여 한 번에 한 원소씩 출력가능

 

탐색

  • 순차탐색 : 주어진 목표 값을 찾아 리스트를 앞에서 부터 순차적으로 탐색

    목표가 발견된 첫 번째 위치를 리턴, 발견되지 않으면 음수를 리턴

  • 이진탐색 : 주어진 배열이 정렬되어 있을 때 주어진 배열을 반복적으로 반으로 나누어 가며,

    각 단계에서 그 범위에 목표를 포함하는 반쪽에 집중

 

java.util.Arrays 클래스

배열 조작을 위한 여러 유틸리티 메소드를 제공, 모두 static으로 선언됨

주요메소드

  • binarySearch(double[] a, double x)
  • equals(double[] a, double[] b)
  • fill(double[] a, double x)
  • fill(double[] a, int lo, int hi, double x)
  • sort(double[] a)

 

사용자 정의 IntArrays 클래스

int배열을 위한 "집에서 만든(homegrown)" IntArrays 클래스: 리스팅6

 

순서배열의 유지

정렬된 배열에 새 원소 삽입 : 배열 구현의 경우 새 원소보다 큰 원소들을 모두 이동시켜야 함

  1. 새 원소에 대한 공간 확보
  2. x를 올바론 위치에 복사

 

간접참조

  • 원소의 실제 위치를 추적하기 위한 보조 인덱스 배열 사용

    ㄴ 추가의 공간(두번째 배열)을 필요로 함

    ㄴ 원소를 이동시킬 필요 없음

  • 인덱스 배열의 사용

    ㄴ 원소의 삽입과 삭제 시 인덱스 배열 원소들의 이동 필요

  • 개선안

    ㄴ 데이터 배열에서 사용하는 것과 동일한 배열 위치를 인덱스 배열에 사용하도록 함

    ㄴ 인덱스 배열은 데이터 배열의 원소 순서를 추적

    ㄴ 추가적인 변수는 인덱스 배열과 데이터 배열 양쪽의 자유 위치의 인덱스를 저장
