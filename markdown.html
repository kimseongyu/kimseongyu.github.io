TOPCIT 기술영역 프로그래밍 언어와 개발환경

프로그래밍 언어의 개요

  • 프로그래밍 언어 개념

    • 저급언어 : 기계 중심적인 언어로 실행 속도가 빠르고 기계마다 다른 코드를 가짐
    • 고급언어 : 사람이 이해하기 위운 자연어에 가깝게 만들어짐

  • 인터프리터 언어 : 중간과정 없이 원시 프로그램을 직접 저급언어로 바꾸면서 동시에 실행

    • 장점 : 완전한 기계어 번역을 기다리지 않고 필요 시 마다 실행, 원시 언어 형태를 기억하므로 메모리 절약
    • 단점 : 원시 언어 형태를 유지하므로 메모리 절약

  • 컴파일러 언어 : 어느 특정 언어를 동등한 의미를 갖는 다른 언어로 바꾸는 것을 의미

    • 장점 : 번역된 코드 저장 가능, 컴파일 한 후 바로 재실행 가능
    • 단점 : 기계어로 변환하는데 많은 시간 소요, 메모리 낭비 발생 가능

 

주요 프로그래밍 언어의 특징

  • C언어 : 실행속도가 빠르고 효율적인 메모리 관리 용이, 절차지향적(프로그램 실행이 정해진 순서에 따라 진행)
  • C++언어 : C언어에서 객체 지향 프로그래밍을 지원하는 언어로, 캡슐화와 자료 은닉, 상속성과 재사용성, 다형성이 강조됨
  • Java언어 : C++의 복잡성을 단순화, 항상 장동 가비지 콜렉션을 수행
  • Python언어 : 동적 타이핑으로 실행 시간에 자료형을 확인가능, 멀티패러다임 프로그래밍 언어(절차적, 함수형, 객체 지향)
  • JavaScript언어 : 간단하게 코딩을 짜고 프로그래밍 가능, 별도의 컴파일 과정이 필요하지 않아 쉽게 배울 수 있음

 

소프트웨어 개발 프레임워크

  • 소프트웨어 개발 프레임워크 : 효율적인 개발 정보시스템 개발을 위한 코드 라이브러리, 인터페이스 규약, 설정정보 등의 집합

    이점 : 코드 품질 향상, 개발 생산성 증대, 요지보수성 향상, 신규시스템 위험 감소

  • 스프링 프레임워크 : 자바 플랫폼을 위한 오픈소스 어플리케이션 프레임워크, 안정성과 유연성이 입증됨

  • 전자정부 표주프레임워크 : 시스템 개발, 운영 시 필요한 기본 기능들을 표준화하여 미리 구현해 둔 공공사업용 표준개발프레임워크

    특징 : 개방형 표준준수, 사용 솔루션 연계, 국가적 표준화 지향, 변화 유연성

    구성요소 : 실행환경, 개발환경, 운영환경, 관리환경, 공통컴포넌트

 

통합개발환경(IDE)

  • 통합개발환경 : 코딩, 디버깅, 컴파일, 배포 등 프로그램 개발에 관련된 모든 작업을 하나의 소프트웨어 안에서 처리하는 환경을 제공하는 개발용 소프트웨어

    구성요소 : 편집기, 빌드도구, 디버거, 프로젝트 관리

  • CI(Continuous Integration) : 여러 명으로 구성된 팀이 작업한 결과를 자주 통합하는 소프트웨어 개발 활동

  • 소프트웨어 빌드 : 소스 코드 파일을 컴퓨터에서 실행할 수 있는 독립 소프트웨어 가공물로 변환하는 가정

  • 일일 빌드 : 매일 전체적으로 다시 컴파일하고, 기본 동작을 검증하기 위해 필요한 일련의 테스트를 거치는 공정

  • 소프트웨어 배포 : 소프트웨어 시스템을 만드는 모든 행위를 지칭
