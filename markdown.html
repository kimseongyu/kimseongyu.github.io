컬렉션

컬렉션이란?

자바에서 자료구조를 구현한 클래스, java.util 패키제이 존재

컬렉션의 종류

  • Collection : 모든 자료구조의 부모 인터페이스로서 객체의 모임을 나타냄
  • Set : 집합(중복된 원소를 가지지 않는)을 나타내는 자료구조
  • List : 순서가 있는 자료구조로 중복된 원소를 가질 수 있음
  • Map : 키와 값들이 연결되어 있는 사전과 같은 자료구조
  • Queue : 들어온 순서대로 나가는 자료구조

 

List

  • ArrayList : array와 같은 형태로 타입 매개변수를 가지는 제네릭 클래스로 제공된다.

    add()를 사용하여

  • LinkedList : 각 원소를 링크로 연결하고 각 원소들은 다음 원소를 가리키는 링크를 저장한다.

    add() : 데이터를 위치를 지정하여 삽입할 수 있다.

    set() : 특정한 위치에 있는 원소를 바꿀때 사용한다.

    remove() : 데이터를 삭제할 때 사용한다.

    get() : 저장된 객체를 가져올 때 사용한다.

    indexOf() : 저장된 데이터의 위치를 반환함. 같은 데이터가 여러개 저장되어 있으면 앞에 있는 데이터의 위치가 반환됨.

 

Set

  • HashSet : 해쉬 테이블에 원소를 저장하기 때문에 성능면에서 가장 우수하지만 원소들의 순서가 일정하지 않다.
  • TreeSet : 레드-블랙 트리에 원소를 저장하여 값에 따라 순서가 결정되지만 HastSet보다는 느리다.
  • LinkedHashSet : 해쉬 테이블과 연결리스트를 합친 것으로 원소들의 순서는 삽입되었던 순서와 같다.

 

Queue

  • 우선순위 큐 : 원소들이 무작위로 삽입되었더라도 정렬된 상태로 원소들을 추출한다.

    이진트리의 일종인 heap이라는 자료구조를 내부적으로 사용한다.

    poll(): 큐의 처음에 있는 원소를 가져오고 제거한다.

    peek(): 큐의 처음에 있는 원소를 제거하지 않고 가져온다.

 

Map

  • HashMap : 해싱 테이블에 데이터를 저장

  • TreeMap : 탐색 트리에 데이터를 저장한다.

  • LinkedHashMap : 해쉬 테이블과 연결리스트를 합친 Map이다.

    put()을 사용하여 데이터를 저장하고 entrySet()을 사용하여 저장된 데이터를 방문한다.

 

Collections 클래스

  • 정렬(Sorting) : List 인터페이스를 구현하는 컬렉션에 대하여 정렬을 수행한다.

    배열을 정렬 할때는 asList()를 사용하여 리스트로 변경해야 한다.

  • 섞기(Shuffling) : shuffle() 순서를 무작위로 만든다.
  • 탐색(Searching) : binarySearch()를 사용하여 리스트 안에서 원하는 원소를 찾는다.

 

반복자(iterator)

  • hasNext() : 아직 방문하지 않은 원소가 있으면 true를 반환
  • next() : 다음 원소를 반환
  • remove() : 최근에 반환된 원소를 삭제
