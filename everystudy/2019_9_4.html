컴프2 과제

ArrayList를 직접 구현해보는 과제를 하였다.


public class ArrayList<E> implements List<E>{
    private static final int DEFAULT_CAPACITY = 10;
    private static final Exception IndexOutOfBoundsException = null;
    private Object[] elementData;
    private int size = 0;
             	
    public ArrayList() {
	    this.elementData = new Object[DEFAULT_CAPACITY];
    }
                        	
    public ArrayList(int size) {
	    this.elementData = new Object[size];
    }
                        	
    @Override
    public String toString() {
	    StringBuilder str = new StringBuilder("[");
	    for (int i = 0; i < size; i++) {
		    str.append(elementData[i]);
		    if(i < size -1)
			    str.append(", ");
	    }
	    str.append("]");
	    return new String(str);
    }
                        	
    @Override
    public boolean addLast(E element) {
	    if(size > 0 && this.elementData[size - 1] != null)
		    resize();
	    this.elementData[size++] = element;
	    return true;
    }
                        	
    @Override
    public boolean addFirst(E element) {
	    if(size > 0 && this.elementData[size - 1] != null)
		    resize();
	    return add(0, element);
    }
                        	
    @Override
    public boolean add(int index, E element) {
	    if(size > 0 && this.elementData[size - 1] != null)
		    resize();
	    if(size < index)
		    throw new IndexOutOfBoundsException();
	    for(int i = size - 1; i >= index; i--) {
		    elementData[i + 1] = elementData[i];
	    }
	    elementData[index] = element;
	    size++;
	    return true;
    }
                        	
    @Override
    public E remove(int index) {
	    if(size <= index || index < 0)
		    throw new IndexOutOfBoundsException();
	    Object obj = this.elementData[index];
	    for(int i = index + 1; i <= size - 1; i++) {
		    this.elementData[i - 1] = this.elementData[i];
	    }
	    size--;
	    elementData[size] = null;
	    return (E)obj;
    }
                        	
    @Override
    public E removeFirst() {
	    return this.remove(0);
    }
                        	
    @Override
    public E removeLast() {
	    return this.remove(size - 1);
    }
                        	
    @Override
    public E get(int index) {
	    return (E)this.elementData[index];
    }
                        	
    @Override
    public int size() {
	    return this.size;
    }
                        	
    @Override
    public int indexOf(E element) {
	    for(int i = 0; i < size; i++) {
		    if(element.equals(elementData[i]))
			    return i;
	    }
	    return -1;
    }
                        	
    private void resize() {
	    Object[] obj = this.elementData;
	    this.elementData = new Object[2*size];
	    for(int i = 0; i < size; i++) {
		    this.elementData[i] = obj[i];
	    }
    }
                        	
    public Iterator<E> iterator(){
	    return new ListIterator();
    }
                        	
    public class ListIterator<E> implements Iterator<E>{
	    private int nextIndex;
                        		
	    private ListIterator() {
    	    this.nextIndex = 0;
        }
                        		
        public boolean hasNext() {
            return nextIndex < size();
        }
                        		
        public E next() {
            return (E)elementData[nextIndex++];
    	}
                        		
	}
}

인공지능공부

'케라스 창시자에게 배우는 딥러닝'으로 인공지능을 공부를 할 것이다.

인공지능이란?

보통의 사람이 수행하는 지능적인 작업을 자동화하기 위한 연구 활동

과거의 AI

  • 심볼릭 AI : 명시적인 규칙을 충분하게 만들어 지식을 다루어 인간 지능의 AI를 만드는 것
  • 전문가 시스템 : 전문적인 지식을 정리하고 컴퓨터에 저장하여 일반인도 전문지식을 이용할 수 있도록 하는 것

머신러닝이란?

데이터와 데이터로부터 기대되는 해답을 입력하며 규칙이 출력되는 것

머신 러닝 시스템은 명시적으로 프로그램되는 것이 아니라 훈련되는것이다.

데이터에서 표현을 학습하기

머신 러닝에 필요한 것
  • 입력 데이터 포인트 : 예를 들어 주어진 문제가 음성이면 데이터 포인트는 사운드 파일이다.
  • 기대 출력 : 음성 인식 작업에서는 사람이 사운드 파일을 듣고 옮긴 글
  • 알고리즘의 성능을 측정하는 법 : 알고리즘의 현재 출력과 기대 출력 간의 차이를 결정하기 위해 필요한 것으로 측정값은 알고리즘의 작동 방식을 교정하기 위한 신호로 다시 피드백되면서 학습한다.
머신 러닝과 딥 러닝의 핵심문제

입력 데이터를 기반으로 기대 출력에 가깝게 만드는 유용한 표현을 학습하는 것

딥러닝에서 '딥'이란?

딥러닝이란?

머신 러닝의 특정한 한 분야로서 연속된 층에서 점진적으로 의미 있는 표현을 배우는데 강점을 가지며 데이터로부터 표현을 학습하는 새로운 방식이다.

'딥'이란?

데이터로부터 모델을 만드는 데 얼마나 많은 층을 사용했는지에 대한 깊이로 '층 기반 표현 학습'또는 '계층적 표현 학습'이라고 부를 수 있다.

신경망이란?

기본 층을 겹겹이 쌓아 올려 구성한 것을 말한다.

딥러닝 알고리즘으로 학습된 표현

최종 출력에 대해 점점 더 많은 정보를 가지지만 원본 이미지와는 다른 숫자 이미지로 변환된다.

딥러닝의 작동원리 이해하기

  1. 데이터가 처리되는 상세 내용은 이련의 숫자로 이루어진 층의 가중치에 저장되어 있으며 각 층은 가중치를 파라미터로 가지는 함수로 표현됨.
  2. 손실 함수를 통해 출력이 기대하는 것보다 얼마나 벗어났는지를 측정하여 기대값과의 차이를 점수로 계산한다.
  3. 역전파 알고리즘을 구현한 옵티마이저로 가중치 값을 수정한다.
  4. 이러한 훈련을 반복하여 결과를 낸다.

컴퓨터 프로그래밍2 5주차 이론 예습

연결리스트를 이용한 리스트 구현

  1. 연결노드 : 인덱스 배열과 데이터 배열의 상대적인 위치는 일치하므로 이들을 데이터-주소쌍의 단일 배열로 표현

    원소에 참조를 위한 객체 사용 : Node 객체의 시퀀스로 생각 가능(배열 대신에 단일 start 참조만을 유지)

    
Node start = new Node(22); //start의 초기화
start.next = new Node(11); //노드의 추가
start.next.next = new Node(33);
                                    
    
start = new Node(22);
Node p = start; //변수 p(포인터)를 사용하여 노드를 순회
p.next = new Node(11);
p = p.next;
p.next = new Node(33);
p = p.next;
                                    
    
for(Node p = start; p != null; p = p.next) //연결 리스트를 이용한 루프문

  1. 연결리스트에 대한 원소 삽입 : 삽입 과정의 단순화를 위해 2-인자 노드 생성자 추가
    (노드의 생성과 삽입을 한꺼번에 수행할 수 있도록 해줌)
    삽입과정
    1. 새로운 노드 앞에 놓일 리스트 노드 p 발견
    2. 새로운 노드를 생성해 부착

  1. 리스트의 앞에 삽입
    문제 : 첫번째 노드로 삽입 시 새로운 노드 앞에 나올 노드가 결여되어 잘못된 위치에 삽입됨
    해결방법
    1. 연결 리스트의 구조를 변경하여 첫 번째 데이터노드 앞에 빈 헤드 노드를 유지
    2. 메소드를 수정하여 특별한 경우를 별도로 처리하도록 한다.

  1. 정렬된 연결리스트에서의 삭제
    delete()메소드 : 1. 원소발견 2. 삭제

    • 중첩 클래스 : 다른 클래스가 멤버인 클래스
      클래스 B가 클래스 A 내부에서만 사용될 때 B는 A 안에 중첩되어야 한다.
