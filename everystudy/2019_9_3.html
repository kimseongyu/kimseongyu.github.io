컬렉션

컬렉션이란?

자바에서 자료구조를 구현한 클래스, java.util 패키제이 존재

컬렉션의 종류

  • Collection : 모든 자료구조의 부모 인터페이스로서 객체의 모임을 나타냄
  • Set : 집합(중복된 원소를 가지지 않는)을 나타내는 자료구조
  • List : 순서가 있는 자료구조로 중복된 원소를 가질 수 있음
  • Map : 키와 값들이 연결되어 있는 사전과 같은 자료구조
  • Queue : 들어온 순서대로 나가는 자료구조

 

List

  • ArrayList : array와 같은 형태로 타입 매개변수를 가지는 제네릭 클래스로 제공된다.
  • LinkedList : 각 원소를 링크로 연결하고 각 원소들은 다음 원소를 가리키는 링크를 저장한다.

    • add() : 데이터를 위치를 지정하여 삽입할 수 있다.
    set() : 특정한 위치에 있는 원소를 바꿀때 사용한다.
    remove() : 데이터를 삭제할 때 사용한다.
    get() : 저장된 객체를 가져올 때 사용한다.
    indexOf() : 저장된 데이터의 위치를 반환함. 같은 데이터가 여러개 저장되어 있으면 앞에 있는 데이터의 위치가 반환됨.

 

Set

  • HashSet : 해쉬 테이블에 원소를 저장하기 때문에 성능면에서 가장 우수하지만 원소들의 순서가 일정하지 않다.
  • TreeSet : 레드-블랙 트리에 원소를 저장하여 값에 따라 순서가 결정되지만 HastSet보다는 느리다.
  • LinkedHashSet : 해쉬 테이블과 연결리스트를 합친 것으로 원소들의 순서는 삽입되었던 순서와 같다.

 

Queue

  • 우선순위 큐 : 원소들이 무작위로 삽입되었더라도 정렬된 상태로 원소들을 추출한다.
    이진트리의 일종인 heap이라는 자료구조를 내부적으로 사용한다.
    poll(): 큐의 처음에 있는 원소를 가져오고 제거한다.
    peek(): 큐의 처음에 있는 원소를 제거하지 않고 가져온다.

 

Map

  • HashMap : 해싱 테이블에 데이터를 저장
  • TreeMap : 탐색 트리에 데이터를 저장한다.
  • LinkedHashMap : 해쉬 테이블과 연결리스트를 합친 Map이다.

    • put()을 사용하여 데이터를 저장하고 entrySet()을 사용하여 저장된 데이터를 방문한다.

 

Collections 클래스

  • 정렬(Sorting) : List 인터페이스를 구현하는 컬렉션에 대하여 정렬을 수행한다.
    • 배열을 정렬 할때는 asList()를 사용하여 리스트로 변경해야 한다.
  • 섞기(Shuffling) : shuffle() 순서를 무작위로 만든다.
  • 탐색(Searching) : binarySearch()를 사용하여 리스트 안에서 원하는 원소를 찾는다.

 

반복자(iterator)

  • hasNext() : 아직 방문하지 않은 원소가 있으면 true를 반환
  • next() : 다음 원소를 반환
  • remove() : 최근에 반환된 원소를 삭제

과제1

                            
package lab03;
import java.util.*;

class StringOneHot{
	private String str;
	Set strSet = new LinkedHashSet<>();
	Map strMap = new HashMap();
	Map> strOneHotVec = new HashMap>();
	public StringOneHot(String str) {
		this.str = str;
		for(int i = 0; i < str.length(); i++) {
			strSet.add(str.substring(i, i+1));
			
		}
		Iterator it = strSet.iterator();
		int key = 0;
		while(it.hasNext()) {
			strMap.put(key, it.next());
			key++;
		}
		Iterator that = strSet.iterator();
		int k = 0;
		while(that.hasNext()) {
			ArrayList list = new ArrayList();
			list.clear();
			for(int j = 0; j < strSet.size() ;j++) {
				list.add(0);
			}
			list.set(k, 1);
			strOneHotVec.put(that.next(),list);
			k++;
		}
		
	}
	public String getString() {
		return str;
	}
	public Set getStrSet(){
		return strSet;
	}
	public Map getStrMap(){
		return strMap;
	}
	public Map> getStrOneHotVec(){
		return strOneHotVec;
	}
}

public class StringOneHotTest {
	public static void main(String[] args) {
		String str = "Hello";
		StringOneHot soh = new StringOneHot(str);
		System.out.println("문자열: " + soh.getString());
		System.out.println("문자열 Set: " + soh.getStrSet());
		System.out.println("문자열 Map: " + soh.getStrMap());
		System.out.println("문자열 One-hot Vector: " + soh.getStrOneHotVec());
		
		System.out.println("One-hot Vector로 나타낸" + soh.getString());
		for(int i = 0; i < str.length(); i++) {
			System.out.println(soh.getStrOneHotVec().get(str.substring(i, i+1)));
		}
	}
}

package lab03;
import java.util.*;

public class GradeTOScore {
	public static void main(String[] args) {
		Scanner input = new Scanner(System.in);
		ArrayList gradeList = new ArrayList<>();
		Map changegrade = new HashMap<>();
		changegrade.put("A",4.0);
		changegrade.put("B",3.0);
		changegrade.put("C",2.0);
		changegrade.put("D",1.0);
		changegrade.put("F",0.0);
		String grade;
		while(true) {
			System.out.print("학점을 입력하세요(A, B, C, D, F):");
			input.hasNextLine();
			grade = input.nextLine();
			if(grade.length() == 0) { break; }
			if(64 < grade.toUpperCase().charAt(0) && grade.toUpperCase().charAt(0)< 71 && grade.length() == 1) {
				if(grade.toUpperCase().charAt(0) != 69) {
					gradeList.add(grade.toUpperCase());
				}else {
					System.out.println("잘못된 학점 입력입니다.");
				}
			}else {
				System.out.println("잘못된 학점 입력입니다.");
			}
		}
		System.out.println("------------------------------");
		gradeList = new ArrayList<>(new HashSet<>(gradeList));
		for(String s : gradeList) {
			System.out.println("저장된 학점:" + s + " 변환한 점수: " + changegrade.get(s));
		}
	}
}

AI Good & Bad

사람들은 AI Fantasy를 가지고 있다. 하지만 현실을 보자면 AI는

인간이 할 수 있는 것에 대한 정의가 없음 , AI의 연료인 데이터가 부족 , AI에 대한 이해가 낮음

현재 인공지능이 잘하는것(2019)

말로 설명할 수 없는 그 무언가를 표현하는 것 을 기반으로 해서

  • 정해진 답 중 하나를 고르는 것 / classification ex) 사람 수 인식

    ㄴ> 1.예제문제를 풀게하고 2. 정답과 비교하여 3. 잘 맞추는 방향으로 학습

  • 사람이 가보지 못한/못할 길을 먼저 가보는 것 /강화학습

    ㄴ> 한 로봇이 배운 것을 다른 로봇에게 전수하여 더 빠르게 학습 가능

  • 사람이 구분하지 못할 정도의 유사 데이터 생성 / generative adversarial network

  • 모르는 것을 모른다고 말하는것
  • 결정에 대한 논리적 설명능력
  • 사람들 사이에서도 정답이 없는 문제

 

Decision Making Frames

인간 대 인공지능이 아닌 더 나은 인간을 만들 수 있도록 인공지능을 개발

    인간이 잘하는 분야에 인공지능을 하자고 하면 지는게임이거나 극단으로 가서 할만한 게임

    AI가 문제를 풀때 전문가에게 90%이상의 동의를 받을 때 정답

  3. benefit from using AI with clear story
